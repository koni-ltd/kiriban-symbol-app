# キリバン報告機能仕様

## 概要

本機能は、90年代の個人サイト文化に存在した「キリ番報告」を再現する機能です。

通常の掲示板とは異なり、投稿データをアプリケーションのデータベースに保存する仕組みではありません。

本機能では **Symbol ブロックチェーンの送金履歴を BBS 風に表示**します。
ユーザーが管理人ウォレットへ XYM を送金し、その **トランザクションメッセージを掲示板の投稿として扱います**。

つまり本機能は次の構造です。

- 投稿 = Symbol送金トランザクション
- メッセージ = トランザクションメッセージ
- 投稿一覧 = 管理人ウォレット宛送金履歴

このため、掲示板のような **投稿保存処理は存在しません**。

---

## 目的

90年代個人サイトの以下の文化を再現することを目的とします。

- キリ番を踏んだ人が自己申告する
- 管理人へ報告する
- 掲示板に記念カキコが残る

これを以下の構成で実現します。

1. ユーザーがキリ番だと判断する
2. Symbolウォレットから管理人ウォレットへ送金する
3. メッセージを書いて送金する
4. 送金履歴をサイトが読み取る
5. BBS風に表示する

---

## 機能の前提

本機能は以下の前提で設計されています。

- サイト側ではキリ番判定を行わない
- キリ番はユーザーの自己判断
- 投稿フォームは存在しない
- 送金トランザクションが投稿に相当する

したがって以下は行いません。

- 本当にキリ番だったかの検証
- キリ番番号の記録

---

## データソース

データは **Supabase ではなく Symbol ブロックチェーン**から取得します。

取得対象

- 管理人ウォレット宛の送金履歴

対象トランザクション

- confirmed transaction
- transfer transaction

---

## 抽出条件

最小実装では以下の条件で一覧を生成します。

**条件**

- 宛先が管理人ウォレット
- confirmed transaction
- transfer transaction

**並び順**

- 新しい順

**メッセージ**

- message が空でも表示
- message が空の場合は `"あぼーん"` と表示

---

## 表示項目

一覧には以下を表示します。

|項目|内容|
|---|---|
|投稿日|transaction timestamp|
|ID|送信元ウォレットアドレス|
|送金額|XYM数量|
|メッセージ|transaction message|

---

## 表示仕様

### 投稿日時

Symbol transaction timestamp を人間可読形式に変換して表示します。

例

```
YYYY-MM-DD HH:mm:ss
```

---

### ウォレットアドレス

送信元ウォレットアドレスを表示します。
BBS の **ID** に相当します。

---

### 送金額

送信された XYM 数量を表示します。

例

```
1 XYM
0.5 XYM
12.345 XYM
```

---

### メッセージ

送金時のトランザクションメッセージを表示します。

注意

- HTMLとして解釈しない
- 必ずプレーンテキスト表示

message が空の場合

```
あぼーん
```

を表示します。

---

## UI仕様

見た目は **BBS風**です。

表示例

```
5: [TALICE...WALLET] 2026-03-15 13:46
1 XYM
キリ番踏みました！
```

ただし実態は掲示板ではなく、
**送金履歴ビューです。**

---

## 実装しない機能

以下は本仕様では実装しません。

- 投稿用DB
- 投稿API
- 投稿フォーム
- 管理画面
- 投稿編集
- 投稿削除
- キリ番判定
- キリ番番号管理

---

## 設計上の注意

本機能は掲示板ではありません。

投稿行為は

```
Symbol送金
```

です。

そのため **Supabase に投稿を保存すると設計が破綻します。**

理由

- オンチェーン履歴とDBの二重管理
- 整合性問題
- 仕様の意図から逸脱

---

## 盲点

「キリバン一覧」という名称から
**キリ番番号が必要と誤解されやすい**ですが、

今回の仕様では

```
キリ番番号は存在しません
```

理由

- サイトがキリ番判定をしない
- ユーザーの自己申告文化を再現するため

番号を持たせると整合性が崩れます。

---

## 最終仕様

キリバン一覧は以下の仕様です。

- Symbol送金履歴を取得
- Supabaseには保存しない
- 表示項目
  - 投稿日時
  - ウォレットアドレス
  - 送金額
  - メッセージ
- 新しい順表示
- message 空は `"あぼーん"`
- 見た目はBBS
- キリ番判定なし

---

## 実装要件（要約）

キリバン一覧は、

> 管理人ウォレット宛の confirmed transfer transaction を取得し
> 送信元アドレス / 送金額 / メッセージ / 日時 を
> BBS風に新着順表示する機能である。

投稿データは **Supabase に保存しない**。
キリ番判定も行わない。
